# Creative SOP - How to Use the Content Production System

## Overview
This system manages all content across 4 channels: Personal Media AM Posts, Personal Media PM Posts, Company Social Media, and YouTube. Each channel has its own Instructure file (planning sheet) and delivery folder.

## Content Types
Every piece of content must be one of these 5 types:
1. **Talking Reel** - Direct-to-camera talking content
2. **BTS Reel** - Behind-the-scenes footage with voiceover or text
3. **Voice-Over Reel** - B-roll footage with recorded voiceover
4. **Carousel** - Multi-slide image post
5. **Client Result** - Testimonial, transformation, or case study content

## Workflow Step-by-Step

### Step 1 - [Creative Director] Plans Content
- Open the relevant Instructure file for the channel
- Fill in: Type, Hook, Topic, Caption, CTA for each day
- Write full scripts in the weekly script files (2026/[Month]/[Week]/)
- Target: complete creative 7 days before posting date

### Step 2 - [Content Manager] Schedules
- Assign dates to each content row
- Confirm times match the channel schedule (AM 09:00, PM 10:30, Company 12:00, YouTube 18:00)
- Send scripts to editing team

### Step 3 - Editing Team Creates
- Pull scripts from the weekly script files
- Pull assets from the Asset Bank (Image-Bank/, Video-Bank/, Client-Result-Bank/)
- Create the final content piece (video edit, carousel design, etc.)
- Follow editor notes in the script file for specific direction

### Step 4 - [Content Manager] QA
- Check hook matches the script
- Check caption is complete and formatted correctly
- Check CTA is correct and ManyChat keyword is live (if applicable)
- Mark "Edited" checkbox when editing is complete
- Mark "Checked" checkbox when QA passes

### Step 5 - [Content Manager] Distributes
- Upload final content to the correct Google Drive delivery folder
- Post or schedule on the platform
- Mark "Posted" or "Scheduled" checkbox
- Log views after 7 days

### Step 6 - [Automation Lead] (where applicable)
- Set up ManyChat keywords before content goes live
- Test DM automation flows
- Confirm triggers are working

## File Naming Conventions
- Scripts: Named by week (e.g. AM-Scripts-W14.md)
- Images: [context]-[number].jpg (e.g. studio-001.jpg)
- Videos: [date]-[description].mp4 (e.g. 2026-04-07-bts-filming.mp4)
- Client results: [client-name]-[type]-[date] (e.g. sarah-testimonial-2026-03.png)

## Quality Checklist (Before Posting)
- [ ] Hook is attention-grabbing and matches the script
- [ ] Caption flows naturally, no full stops, line breaks between thoughts
- [ ] CTA is clear and actionable
- [ ] ManyChat keyword is set up (if applicable)
- [ ] Content type matches the Instructure plan
- [ ] Correct assets used (no repeated images within the same month)
- [ ] Posted to correct platform at correct time
